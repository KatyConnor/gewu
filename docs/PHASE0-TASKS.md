## 1. 阶段目标

验证所有高、中风险项，确认技术可行性，为后续开发提供决策依据。

### 1.1 高风险项 (必须验证)

| 序号 | 风险项 | 验证内容 | 决策点 |
|------|--------|----------|--------|
| R-01 | Firecracker 兼容性 | KVM 支持、平台兼容 | 决定沙箱方案 |
| R-02 | 自研网关性能 | QPS、延迟、并发 | 决定是否自研 |
| R-03 | OceanBase 性能 | 查询延迟、吞吐量 | 决定默认数据库 |

### 1.2 中风险项 (需要验证)

| 序号 | 风险项 | 验证内容 | 决策点 |
|------|--------|----------|--------|
| R-04 | 工作流引擎复杂度 | 状态机、并行流程 | 决定是否自研 |
| R-05 | SSE 集群模式 | 跨节点同步、故障转移 | 决定通信方案 |
| R-06 | DragonflyDB 兼容性 | Redisson 兼容、性能 | 决定缓存方案 |

## 6. 决策矩阵

### 6.1 沙箱方案决策

| 条件 | 决策 |
|------|------|
| Firecracker 可用且性能达标 | 使用 Firecracker |
| Firecracker 不可用但 gVisor 可用 | 使用 gVisor |
| Firecracker 和 gVisor 都不可用 | 使用 Docker |

### 6.2 API 网关决策

| 条件 | 决策 |
|------|------|
| 自研网关 QPS > 10,000 且延迟 < 100ms | 继续自研 |
| 自研网关性能不达标 | 评估 Spring Cloud Gateway |
| 自研网关开发复杂度不可控 | 评估开源网关 |

### 6.3 数据库决策

| 条件 | 决策 |
|------|------|
| OceanBase 性能达标且团队可接受 | 使用 OceanBase |
| OceanBase 性能不达标或团队不可接受 | 使用 MySQL |
| OceanBase 和 MySQL 都不满足 | 评估其他数据库 |

### 6.4 缓存方案决策

| 条件 | 决策 |
|------|------|
| DragonflyDB 兼容性良好且性能达标 | 使用 DragonflyDB |
| DragonflyDB 兼容性有问题 | 使用 Redis |
| DragonflyDB 和 Redis 都不满足 | 评估其他缓存 |
