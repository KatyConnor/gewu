## 1. 架构概述

格物平台是一个 **AI 驱动的智能开发协作平台**，采用分层架构 + DDD（领域驱动设计）模式，支持多数据库、多缓存、多消息队列，满足信创合规要求。

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| **分层架构** | 表现层、应用层、领域层、基础设施层清晰分离 |
| **DDD** | 领域驱动设计，业务逻辑封装在领域层 |
| **多数据库支持** | OceanBase/MySQL/PostgreSQL/Oracle |
| **多缓存支持** | DragonflyDB/Redis |
| **多消息队列** | RocketMQ/Kafka/RabbitMQ |
| **信创合规** | 100% 满足信创目录要求 |
| **自主可控** | 自研 API 网关、自研工作流引擎 |

## 11. 技术决策记录 (ADR)

### ADR-001: 选择分层架构 + DDD

**状态**: 已批准

**决策**: 采用分层架构 + DDD 模式，业务逻辑封装在领域层。

**理由**:
- 清晰的分层结构，职责明确
- DDD 提升业务建模能力
- 易于测试和维护

---

### ADR-002: 选择 OceanBase 作为主数据库

**状态**: 已批准

**决策**: 选择 OceanBase 作为主数据库。

**理由**:
- 分布式数据库，支持海量数据
- MySQL/PostgreSQL/Oracle 兼容
- 信创合规
- 性能优异

---

### ADR-003: 自研 API 网关

**状态**: 已批准

**决策**: 自研 API 网关 (Java/Netty)。

**理由**:
- 信创合规要求
- 自主可控
- 性能可定制
- 功能可扩展

---

### ADR-004: 选择 Firecracker 作为沙箱运行时

**状态**: 已批准

**决策**: 选择 Firecracker 作为沙箱运行时。

**理由**:

- 轻量级，开销小
- 性能优异
