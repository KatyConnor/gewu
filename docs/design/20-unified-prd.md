## 1. 产品概述

### 1.1 产品愿景

格物平台旨在成为企业级 **AI 驱动的智能开发协作平台**，覆盖从需求分析、设计、开发、测试到部署的全流程。通过融合 OpenCode 的 AI 编码代理能力和格物平台的企业协作体系，赋能软件开发全生命周期，使团队协作效率提升 50%+，编码效率提升 40%+，线上缺陷率降低 60%+。

> **差异分析 1.1**: OpenCode 定位为"单人 AI 编码代理工具"，格物定位为"企业级 AI 协作平台"。合并后产品同时覆盖单人深度编码场景和企业多角色协作场景。OpenCode 的 Session 会话体系、Tool 执行引擎、MCP 集成范式将被吸收为格物的 Agent 系统核心能力。MCP 协议因信创合规将替换为自研工具协议，但 OpenCode 的连接管理、工具发现、OAuth 等设计思想完整保留。

### 1.2 产品目标

- **统一协作**: 覆盖 8 种角色（架构师、产品经理、前后端开发、测试、运维、安全、技术负责人、开发者-通用），提供统一的协作平台
- **AI 增强**: 吸收 OpenCode 的 Agent 系统设计，将 AI 能力深度嵌入开发流程，提供智能对话、代码补全、自动化审查、智能测试生成、会话分享、版本管理
- **安全合规**: 100% 满足信创要求，支持国密算法（SM2/SM3/SM4）、国产数据库、国产中间件
- **高性能**: 支持企业级大规模团队（500+ 并发用户）的实时协作，核心 API P95 < 200ms，非核心 API P95 < 500ms（详见 21-unified-architecture.md §9.3 性能 SLA）
- **可扩展**: 灵活的工作流引擎、工具注册系统和配置热重载，支持企业自定义流程

### 1.3 核心能力矩阵

| 能力域 | MVP (Phase 1) | Phase 2 | Phase 3+ |
|--------|--------------|---------|----------|
| 用户管理 | 注册登录 + RBAC + 国密 | OAuth 2.0 SSO | 多租户 |
| 项目管理 | CRUD + 成员管理 | 统计报表 | 敏捷看板 |
| 会话/消息 | 创建+发送+SSE+历史+Part表+自动标题 | 富媒体消息 + 已读 + @提及 | AI 摘要 + 会话版本管理 |
| Agent 系统 | Agent 配置 + 工具系统 + 权限评估 + 执行引擎 + 超时/结果限制 | Agent 编排 + 执行历史 | Agent 市场 |
| 工作流引擎 | 状态机核心 | 可视化设计器 | 自动化触发 |
| 沙箱隔离 | Docker 沙箱 | gVisor 集成 | Firecracker |
| 代码管理 | Git 集成 | 代码审查 | AI 代码审查 |

| API 网关 | 路由 + JWT 认证 + 国密 | 限流 + 协议转换 | 服务网格 |

## 6. 状态流转（从 OpenCode 吸收）

### 6.1 会话状态

```
stateDiagram-v2
    [*] --> 创建
    创建 --> 活跃: 发送消息
    活跃 --> 空闲: 停止活动
    空闲 --> 活跃: 继续对话
    活跃 --> 压缩: 上下文过长
    压缩 --> 活跃: 压缩完成
    活跃 --> 归档: 用户归档
    归档 --> [*]
    归档 --> 活跃: 取消归档
```

| 状态 | 说明 | 吸收状态 |
|------|------|---------|
| 创建 | 会话刚被创建，尚未发送消息 | ✅ 新增状态 |
| 活跃 | 会话正在被使用，AI 正在响应或用户正在输入 | ✅ 新增状态 |
| 空闲 | 会话暂时无活动 | ✅ 新增状态 |
| 压缩 | AI 上下文过长，正在进行上下文压缩 | ✅ 新增状态（AI 会话特有） |
| 归档 | 会话已归档，不再活跃列表中显示 | ✅ Gewu 已有 |

> **差异分析 6.1**: OpenCode 的会话压缩状态是 AI 编码代理的特有能力——当 AI 上下文窗口接近上限时自动压缩。Gewu 原会话管理没有此概念，合并后保留。

### 6.2 工具执行状态

```
stateDiagram-v2
    [*] --> 待执行
    待执行 --> 权限检查: 需要权限
    权限检查 --> 待执行: 权限拒绝
    权限检查 --> 执行中: 权限允许
    待执行 --> 执行中: 无需权限
    执行中 --> 完成: 执行成功
    执行中 --> 失败: 执行失败
    执行中 --> 超时: 执行超时 (30s)
    完成 --> [*]
    失败 --> [*]
    超时 --> [*]
```

| 状态 | 触发条件 | 说明 |
|------|---------|------|
| 待执行 | 工具调用被 Agent 触发 | 初始状态 |
| 权限检查 | 工具标记为"危险"（文件写/Shell/网络） | 需用户授权 |
| 执行中 | 权限允许或无权限要求 | 工具正在执行 |
| 完成 | 工具执行成功并返回结果 | 终态 |
| 失败 | 工具执行过程异常 | 终态 |
| 超时 | 超过 timeout_ms（默认 30s） | 终态，结果截断 |

### 6.3 自研工具协议连接状态（替代 MCP 状态）

```
stateDiagram-v2
    [*] --> 断开
    断开 --> 连接中: 发起连接
    连接中 --> 已连接: 连接成功
    连接中 --> 失败: 连接失败
    连接中 --> 需要认证: 认证失败
    已连接 --> 断开: 连接断开
    已连接 --> 需要认证: 认证过期
    需要认证 --> 连接中: 重新认证（SM2）
    失败 --> 断开: 重置
```

> **差异分析 6.3**: 保持 OpenCode MCP 连接的完整状态机设计，仅将认证方式从 OAuth 2.0 替换为 SM2 国密认证。
